# Payments System — Django Interview

## Описание
Проект представляет собой backend-сервис для обработки webhook-ов от банка. Сервис принимает платежи, начисляет баланс организациям по ИНН и предоставляет API для получения текущего баланса и истории изменений баланса.

## Технологии
- Python 3.9
- Django 4.2.17
- MySQL (в продакшене) / SQLite (для разработки)

## API Endpoints

### 1. Вебхук от банка (POST)
**URL:** `/api/webhook/bank/`  
**Метод:** POST  
**Content-Type:** application/json  
**Тело запроса:**
```json
{
  "operation_id": "ccf0a86d-041b-4991-bcf7-e2352f7b8a4a",
  "amount": 145000,
  "payer_inn": "1234567890",
  "document_number": "PAY-328",
  "document_date": "2024-04-27T21:00:00Z"
}
```
**Ответ:**
- 200 OK: `{"status": "success"}` или `{"status": "already processed"}`
- 400 Bad Request: `{"error": "..."}`

### 2. Получение баланса организации (GET)
**URL:** `/api/organizations/<inn>/balance/`  
**Метод:** GET  
**Ответ:**
- 200 OK: `{"inn": "1234567890", "balance": 145000}`
- 404 Not Found: `{"error": "Organization not found"}`

### 3. Получение истории изменений баланса (GET)
**URL:** `/api/organizations/<inn>/balance/history/?limit=10`  
**Метод:** GET  
**Параметры:**
- `limit` (опционально, по умолчанию 10, максимум 100) — количество последних изменений
**Ответ:**
- 200 OK: 
```json
{
  "inn": "1234567890",
  "history": [
    {"amount": 100.0, "created_at": "2024-04-27T21:00:00Z", "balance_after": 145100.0},
    ...
  ]
}
```
- 404 Not Found: `{"error": "Organization not found"}`

## Бизнес-логика и защита данных
- Все критические операции (создание платежа, логирование, изменение баланса) выполняются внутри атомарной транзакции (`@transaction.atomic`).
- Для предотвращения гонки условий используется блокировка записей через `select_for_update()` и атомарные выражения `F()`.
- Проверка на дублирование платежа происходит с блокировкой.
- Изменение баланса происходит явно в сервисе, а не в сигналах или методе `save()`.
- Запись о событии (лог) создаётся до изменения баланса.
